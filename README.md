## **Bug #1 - Mobile Navigation Not Opening**
**Problem:**  
The mobile menu button does nothing when tapped.

**Cause:**  
The event listener was attached to the menu instead of the button.

**Fix:**  
Attach the listener to the correct button and toggle visibility properly.

**Demo:**  
- Broken version  
- Fixed version

## **Bug #2 - Form Validation Not Working**
**Problem:**  
The form submitted even when required fields were empty.

**Causes:**  
- Assignment operator used instead of comparison (`=` instead of `===`)  
- Missing `preventDefault()`  
- Validation logic not reached  

**Fix:**  
Add correct comparison, prevent default submission, and show clear messages.

**Demo:**  
- Broken version  
- Fixed version  

---

## **Bug #3 - Layout Breaking on Mobile**
**Problem:**  
The layout overflowed horizontally on small screens and required side scrolling.

**Causes:**  
- Fixed container width  
- No wrapping behaviour  
- Cards too wide  

**Fix:**  
Use responsive flexbox rules, wrapping, breakpoints, and flexible sizing.

# ğŸ›  How I Debug  
My debugging workflow:

1. **Reproduce the bug** under the same conditions the user experiences.  
2. **Inspect HTML, CSS, and JS behaviour** using devtools.  
3. **Isolate the smallest possible cause** (selector, event, style, logic, DOM structure).  
4. **Apply a targeted fix**, not a patch that causes new problems.  
5. **Re-test thoroughly** across devices and screen sizes.  
6. **Explain the fix simply**, so the client understands exactly what changed.
